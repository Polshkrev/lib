### Создание
Чтобы созданить новой записи определен конструктор `entry_init`, который принимает объект типа [адреса](/docs/ru-RU/c/path.md) и возвращает указатель на новый объект типа `entry_t`, тип которого установлен в перечисление `FILE_TYPE`, а содержимое представляет собой пустой инициализированный объект типа [списка букв](/docs/ru-RU/c/collections/string_builder.md).
### Функциональность
Чтобы созданить файла определены три метода:
- `entry_touch`: для создания записи типа `FILE_TYPE`.
- `entry_make_directory`: для создания записи типа `DIRECTORY_TYPE`.
- `entry_create`: универсальная обертка над вышеупомянутыми методами, которая вызывается на основе внутреннего типа данных `file_type_t` записи.

Каждый из вышеупомянутых методов может завершиться неудачей, если путь к записи уже существует или если запись не соответствует указанному типу.

Чтобы удаленить записи определен очень похожий набор методов:
- `entry_remove_file`: для удаления записи типа `FILE_TYPE`.
- `entry_remove_directory`: для создания записи типа `DIRECTORY_TYPE`.
- `entry_remove`: универсальная обертка над вышеупомянутыми методами, которая вызывается на основе внутреннего типа данных `file_type_t` записи.

Каждый из вышеупомянутых методов может завершиться неудачей, если путь к записи не существует, если запись не соответствует указанному типу или если тип записи не может быть определен.

Методы создания и удаления реализуются одновременно и используют безопасные для параллельного выполнения сборки мусора.
