# Версия
Это утилит семантического управления версиями. Её можно использовать для хранения объектов управления версиями для флагов функций. Также её можно использовать вместе с модулем флагов для создания профессионально оформленного флага версии.

## Использование

### Конструкция
Множество способов создания объекта версии. По умолчанию используется функция `version_init`. Все функции-конструкторы возвращают указатель на объект версии, выделенный в куче. Функция `version_init` устанавливает строковые свойства нового объекта версии в `NULL`, а каждое из числовых свойств — в 0. Соглашение об именовании функций-конструкторов следующее:
- Функция с ключевым словом "convert" создаст объект с числовыми свойствами, заданными в соответствии с заданными параметрами.
- Функция с ключевым словом "strings" не требует пояснений, поскольку она создаёт каждое строковое свойство объекта версии.
- Все дополнительные свойства, устанавливаемые в конструкторе, именуются явно.

### Модификация
Чтобы установить любого свойства объекта версии каждому свойству назначен определенный сеттер.
Для изменения любого свойства объекта доступны следующие функции:
- Release: Увеличить основной номер объекта и установить его дополнительный номер и номер версии исправления на 0.
- Update: Увеличить дополнительный номер объекта и установить версию исправления объекта на 0, не изменяя основной номер объекта.
- Patch: Увеличить значение исправления объекта, не изменяя основной или дополнительный номера объекта.
- Publish: Установить основной номер объекта на 1. Если объект оценивается как уже опубликованный, возникает ошибка, и значения не изменяются.

### Сравнивать
Каждое числовое свойство объекта имеет функцию сравнения. Для сравнения одного объекта версии с другим доступна функция `version_compare`. Доступна функция `is_public`, используемая в `version_publish`.

### Разрушение
Чтобы освободить любой из созданных объектов версии, необходимо вызвать `version_delete`.
